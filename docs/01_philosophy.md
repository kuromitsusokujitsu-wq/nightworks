# Layer 0 — 理念

## Mission

個人がClaude Codeを駆使し、専門業界の非効率を溶かす。少人数で大企業の生産性を超える。

最初の戦場は **行政書士の補助金業務**。1件20-40時間の手作業に時給¥5,000を切る歪みがある。ここを溶かす。

## Vision（3年）

行政書士の補助金業務における **標準下書きインフラ** になる。確立後、隣接領域へ横展開する：

- 他士業（社労士の助成金、税理士の各種申請）
- 他公募（自治体補助金、官公庁の許認可申請）

横展開の判断基準は「公開情報で要件が構造化でき、独占業務の専門家が顧客になれること」。

## Values（判断に迷ったときの基準）

| # | Value | 具体的な判断への落とし込み |
|---|---|---|
| 1 | **検証優先** | 思い込みで作らない。実地データ（行政書士の生声・課金実績）が常に仮説に勝つ。 |
| 2 | **自動化原理主義** | 同じ手作業を2回見たら仕組み化する。人間の時間は意思決定と検証にのみ使う。 |
| 3 | **顧客の時給を上げることだけが価値** | 行政書士の作業時間を短縮し採択率を上げる。それ以外の機能は作らない。 |
| 4 | **法と倫理は1ミリも越えない** | 行政書士法・特商法・個人情報保護法の境界を越えない。グレーは進まず止まる。X-Styleの行動規範はこの一点で旧来の「グレーはGO」方針を明確に否定する。 |
| 5 | **撤退は早く、固執しない** | 撤退/ピボットラインに数値で抵触したら、感情を排して撤退・転換する。 |

Value 4 と他の目標（売上・速度）が衝突した場合、常に Value 4 が勝つ（CLAUDE.md 第一原則と一致）。
