# Layer 1 — 経営基盤

## 事業定義

行政書士向け **補助金申請書AI下書き支援ツール**（月額サブスク）。行政書士が自分の顧客（事業者）の補助金申請書を作成する際、その下書きを高速・高品質に生成する裏方ツール。

## ターゲット

- 補助金を専業／準専業にする行政書士。全国約5万人中、補助金で食う層は推定 5,000-10,000人。
- 初手対応公募（情報が公開され構造化しやすいもの）：
  - 小規模事業者持続化補助金
  - ものづくり補助金
  - 事業再構築補助金
  - IT導入補助金

## 価格モデル

- 基本：月額サブスク **¥3-5万/事務所**。
- オプション：成功報酬連動（採択時、行政書士が事業者から受け取る報酬の数%）。
- 初手は無料トライアル（最初の2-3事務所は導入事例づくりのため激安/無料も可）。

## 収益目標

6ヶ月以内に **月次利益 ¥50万**。

MRR逆算（APIコスト等の固定費を差し引いた後の利益ベース）：

| モデル | 必要顧客数 | 備考 |
|---|---|---|
| ¥5万/月 | 約12事務所 | 高単価・濃いサポート |
| ¥3万/月 | 約20事務所 | 低単価・標準化重視 |

## KPIツリー

- **北極星指標：MRR（月次経常収益）**
- 先行指標（ファネル）：ヒアリング数 → デモ数 → 無料トライアル数 → 有料転換率 → 月次解約率

`kpi-analyst` が週次でこのツリーを更新する。

## リソース制約

- 稼働：3-4時間/日（1人）
- 初期予算上限：¥5万
- 自動化前提（Claude Code/サブエージェントで運用工数を最小化）

## 撤退 / ピボットライン（数値・必須）

| 経過 | 条件 | アクション |
|---|---|---|
| 8週 | 課金意思を明示した行政書士が0人 | 事業ピボット判断 |
| 16週 | MRR ¥0 | 撤退判断 |

`kpi-analyst` がライン抵触を検知したら、希望的観測を挟まず CEO へ即アラート。

## リスク管理

| リスク | 対応 |
|---|---|
| 行政書士法（独占業務）抵触 | 第一原則として全納品前に compliance-guardian がゲート。顧客は行政書士のみ。 |
| 特商法（B2Bコールド営業） | 送信者表記・配信停止リンクを文面に固定。sales-automator が遵守を強制。 |
| 個人情報保護法 | 公開名簿のみ利用。事業者機密はAPI送信前に最小化/マスキング。削除要請即応。 |
| APIコスト超過 | 月次コスト上限を設定、kpi-analyst が監視。prompt caching でコスト圧縮。 |
| 業界の閉鎖性で初動が遅い | 最初の2-3事務所を激安/無料で導入事例化、紹介連鎖を狙う。 |
